# Naruto Mythos Collection Tracker — Specifica per Web App

**Stato attuale:**
- Progetto migrato da Vite a Create React App (CRA) con TypeScript e Tailwind CSS.
- Routing funzionante con React Router.
- Dashboard visibile e struttura pagine pronta.
- Configurazione Tailwind e PostCSS aggiornata per compatibilità CRA.
- Collegamento a Supabase predisposto tramite variabili ambiente `.env`.
- Struttura file e cartelle conforme alle best practice React.

**Prossimi step:**
- Implementare login e registrazione con Supabase Auth.
- Sincronizzare la collezione utente con Supabase (cards e user_cards).
- Implementare la visualizzazione e gestione delle carte (possessata/mancante, ricerca, filtri, statistiche, ecc.).
- UI mobile-first e accessibilità.
- Deploy su Vercel.

---

## Obiettivo

Creare una web app leggera, veloce e pratica per tenere traccia della collezione **Naruto Mythos Cards**.

L’app deve permettere all’utente di:

- vedere tutte le carte della collezione;
- segnare quali carte possiede;
- segnare quali carte mancano;
- cercare una carta per numero o nome;
- filtrare per stato: tutte, possedute, mancanti;
- visualizzare immagini delle carte;
- vedere la percentuale di completamento;
- consultare una lista rapida delle carte mancanti;
- usare l’app comodamente da smartphone;
- salvare i dati in cloud tramite Supabase;
- pubblicare il frontend su Vercel.

## Struttura database Supabase

### Tabella `cards`

Contiene la lista ufficiale delle carte.

```sql
create table if not exists public.cards (
  id bigint generated by default as identity primary key,
  card_number int not null unique,
  name text not null,
  image_url text,
  rarity text,
  category text,
  set_name text default 'Naruto Mythos',
  created_at timestamptz default now()
);
```

Campi:

| Campo | Tipo | Descrizione |
|---|---|---|
| id | bigint | ID interno |
| card_number | int | Numero carta |
| name | text | Nome ufficiale carta |
| image_url | text | URL immagine |
| rarity | text | Rarità, se disponibile |
| category | text | Categoria/personaggio/tipo |
| set_name | text | Nome set |
| created_at | timestamptz | Data inserimento |

---

### Tabella `user_cards`

Contiene lo stato della collezione per ogni utente.

```sql
create table if not exists public.user_cards (
  id bigint generated by default as identity primary key,
  user_id uuid not null references auth.users(id) on delete cascade,
  card_id bigint not null references public.cards(id) on delete cascade,
  owned boolean not null default false,
  quantity int not null default 1,
  notes text,
  updated_at timestamptz default now(),
  unique(user_id, card_id)
);
```

Campi:

| Campo | Tipo | Descrizione |
|---|---|---|
| id | bigint | ID interno |
| user_id | uuid | Utente Supabase |
| card_id | bigint | Carta collegata |
| owned | boolean | true = posseduta, false = mancante |
| quantity | int | Quantità posseduta |
| notes | text | Note personali |
| updated_at | timestamptz | Ultimo aggiornamento |

## Row Level Security

Attivare RLS.

```sql
alter table public.cards enable row level security;
alter table public.user_cards enable row level security;
```

### Policy lettura carte pubbliche

Tutti gli utenti autenticati possono leggere la lista carte.

```sql
create policy "Authenticated users can read cards"
on public.cards
for select
to authenticated
using (true);
```

### Policy lettura propria collezione

```sql
create policy "Users can read their own collection"
on public.user_cards
for select
to authenticated
using (auth.uid() = user_id);
```

### Policy inserimento propria collezione

```sql
create policy "Users can insert their own collection"
on public.user_cards
for insert
to authenticated
with check (auth.uid() = user_id);
```

### Policy aggiornamento propria collezione

```sql
create policy "Users can update their own collection"
on public.user_cards
for update
to authenticated
using (auth.uid() = user_id)
with check (auth.uid() = user_id);
```

### Policy eliminazione propria collezione

```sql
create policy "Users can delete their own collection"
on public.user_cards
for delete
to authenticated
using (auth.uid() = user_id);
```

## Trigger aggiornamento `updated_at`

```sql
create or replace function public.set_updated_at()
returns trigger
language plpgsql
as $$
begin
  new.updated_at = now();
  return new;
end;
$$;

drop trigger if exists set_user_cards_updated_at on public.user_cards;

create trigger set_user_cards_updated_at
before update on public.user_cards
for each row
execute function public.set_updated_at();
```

## Inizializzazione collezione utente

Al primo login, se l’utente non ha ancora righe in `user_cards`, creare automaticamente lo stato iniziale.

Logica:

1. Recuperare tutte le carte dalla tabella `cards`.
2. Recuperare la lista dei numeri mancanti.
3. Per ogni carta:
   - se il numero è nella lista mancanti → `owned = false`;
   - altrimenti → `owned = true`.
4. Inserire righe in `user_cards`.

## Query principali Supabase

### Recupero carte

```ts
const { data, error } = await supabase
  .from('cards')
  .select('*')
  .order('card_number', { ascending: true });
```

### Recupero collezione utente

```ts
const { data, error } = await supabase
  .from('user_cards')
  .select(`
    id,
    user_id,
    card_id,
    owned,
    quantity,
    notes,
    updated_at,
    card:cards (
      id,
      card_number,
      name,
      image_url,
      rarity,
      category,
      set_name
    )
  `)
  .eq('user_id', userId)
  .order('card_id', { ascending: true });
```

### Aggiornamento stato carta

```ts
const { error } = await supabase
  .from('user_cards')
  .update({ owned: newOwnedValue })
  .eq('id', userCardId);
```
